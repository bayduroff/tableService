XML Processor Service
Сервис для парсинга XML-файла (YML формата), создания динамических таблиц в PostgreSQL и обновления данных. Проект реализован в рамках тестового задания.

Технологии:
- Java 17
- Spring Boot 3
- Maven
- Groovy XmlSlurper (для парсинга XML)
- PostgreSQL 14
- Docker / Docker Compose

Функциональность:
- getTableNames() – возвращает список названий таблиц, найденных в XML (currencies, categories, offers и др.).
- getTableDDL(String tableName) – генерирует SQL-скрипт CREATE TABLE на основе данных из XML (учитывает все записи таблицы).
- updateTable(String tableName) – создаёт таблицу (если её нет) и обновляет/вставляет записи. При изменении структуры XML (появлении новых колонок) выбрасывает исключение.
- updateAll() – вызывает updateTable для всех таблиц из XML.

Требования:
- Java 17 (для локального запуска)
- Maven (для сборки)
- Docker и Docker Compose (для контейнеризированного запуска)
- PostgreSQL (если запускаете локальный, можно использовать Docker‑образ)

Запуск:
1. Локально (только БД через Docker):
- Убедитесь, что PostgreSQL запущен и доступен (например, через Docker)
- Склонируйте репозиторий:
git clone <url>
cd xml-processor-service
- Соберите проект:
mvn clean package
- Запустите приложение:
java -jar target/*.jar
Приложение будет интерактивным с меню действий через локальную консоль. При необходимости можно изменить CommandLineRunner в XmlProcessorApplication на демонстрационную версию.

2. Через Docker Compose (необходимо )
- В корне проекта выполните:
docker-compose up --build
- Будут запущены два контейнера:
postgres – база данных PostgreSQL (порт 5433 на хосте).
app – Spring Boot приложение.
- Приложение автоматически распарсит XML, покажет список таблиц, сгенерирует DDL и выполнит updateAll. Логи будут выведены в консоль.

Особенности:
- Уникальность: столбец vendorCode в таблице offers объявлен UNIQUE (согласно условию).
- Параметры товаров: все вложенные теги <param> собираются в JSON и сохраняются в колонку params_json.
- Динамическая структура: таблицы создаются на основе всех записей в XML; при изменении состава колонок выбрасывается исключение (требование задания).

Примечания:
- XML загружается по URL: https://expro.ru/bitrix/catalog_export/export_Sai.xml.
- Для работы с Groovy XmlSlurper из Java используется динамический вызов метода attributes() через GroovyObject.
- При локальном запуске без Docker убедитесь, что параметры подключения в application.properties соответствуют вашей БД.
